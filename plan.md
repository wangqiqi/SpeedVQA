### 算法设计评审意见（2026-03-29）

依据 `docs/01_设计.md` 与 `speedvqa/models/speedvqa.py` 的实现，对 **算法设计** 的评级与结论如下（区分「学术新颖性」与「面向工程目标的合理性」）。

**分项评分（10 分制）**

| 维度 | 分数 | 说明 |
|------|------|------|
| 问题与任务定义 | 8.5 | ROI + 短问句二分类 VQA，边界清楚；文档明确不适用开放域/生成式，与产品场景一致。 |
| 架构选型（轻量推理） | 8.0 | MobileNetV3（可选 ResNet）+ DistilBERT + 融合 + MLP，与低延迟、可部署目标匹配；文本侧仍是延迟与算力主要来源，属常见取舍。 |
| 多模态交互深度 | 6.0 | 默认 `concat` 合理；配置中的 `attention` 实为两 token 自注意力，**无**图像–文本跨模态注意力，复杂组合推理能力有限。 |
| 训练目标与鲁棒性 | 7.5 | CE / focal、label smoothing、类别权重、可冻结文本编码器等，工程侧较完整。 |
| 学术/方法新颖性 | 5.5 | 双编码器 + 融合 + 分类头为领域常规做法，非新范式。 |

**综合**

- 若以 **是否适合本项目工程目标（ROI 二分类、可导出、低延迟）** 衡量：约 **7.5～8 / 10**（偏 A-）。
- 若以 **论文级创新或 SOTA 多模态结构** 衡量：约 **5.5～6 / 10**（及格偏上）。

**加分点**

- 任务收窄到二分类 ROI-VQA，避免与开放域 VQA 混谈，设计目标一致。
- 融合可配置（concat / attention / bilinear），便于在精度与速度间做实验。
- 视觉全局池化 + 投影、文本 `[CLS]` + 投影，实现简单稳定，利于训练与 ONNX/TensorRT 导出。

**后续可改进方向（若要拉高「算法」维度得分）**

- 在不大改部署故事的前提下，可探索轻量跨模态交互（例如浅层共注意力、或 CLIP 式对齐后再小头分类）。
- 当前设计在 ROI 已裁剪场景下通常够用；若未来需整图上下文，需另做空间建模或区域级特征。
