# 02 使用（全栈流程）

本文档覆盖从 **环境 → 数据 → 训练 → 导出 → 部署推理** 的完整链路，并与当前实现（`speedvqa/cli`、`speedvqa/data/datasets.py` 等）对齐。**架构与设计理念**见 **[01_设计.md](./01_设计.md)**。导出 / 推理的字段级 API 仍以 **`speedvqa/export/README.md`**、**`speedvqa/inference/README.md`** 为准。

---

## 全栈流程概览

| 阶段 | 你要做的事 | 典型命令 / 入口 |
|------|------------|-----------------|
| 环境 | 安装依赖与可编辑包 | `pip install -r requirements.txt && pip install -e .` |
| 数据 | 按下列格式放置 `dataset_path` | 编辑 `data.dataset_path` |
| 配置 | 改 YAML | `speedvqa/configs/default.yaml` 或 `--config` |
| 训练 | 启动 `ConfigurableTrainer` | `python -m speedvqa.cli.onekey_train` |
| 预测 | 单图 + 单问句试跑 | `python -m speedvqa.cli.onekey_predict` |
| 导出 | 检查点 → `.pt` / `.onnx` / `.engine` | `python -m speedvqa.cli.onekey_export` |
| 部署 | 服务或批处理中调用 | `ROIInferencer` |

产物默认落在 **`runs/`** 与导出目录（见下文）。

## 环境与安装

**要求**：Python **≥ 3.10**（见根目录 **`pyproject.toml`** 的 `requires-python`）。

在**仓库根目录**执行（路径示例：`/path/to/SpeedVQA`）：

```bash
cd /path/to/SpeedVQA
pip install -r requirements.txt
pip install -e .
```

- **可编辑安装**：改 `speedvqa/` 内代码后立即生效，适合开发。
- **首次训练/推理**：`transformers` 会从 Hugging Face 拉取 **`model.text.encoder`** 对应权重与分词器，需能访问外网或配置镜像/缓存。
- **ONNX**：导出/推理需安装 `onnx`、`onnxruntime`（及可选 GPU 版 runtime）。
- **TensorRT**：需本机安装 TensorRT、`pycuda` 等与 CUDA 版本匹配的栈；未安装时导出列表中的 `tensorrt` 会不可用。

**控制台命令**（`pip install -e .` 后）：`speedvqa-train`、`speedvqa-predict`、`speedvqa-export`（定义见 **`pyproject.toml`** 的 `[project.scripts]`）。

## 数据准备

数据集根目录 = 配置里的 **`data.dataset_path`**（默认 `./datasets/vqa_dataset`）。其下至少需要 **`images/`** 放图片；标注可通过下列**一种或多种**方式提供，加载时会**合并**所有有效来源（实现见 **`speedvqa/data/datasets.py`** 中 `VQADataset._load_samples`）。

### 目录约定

```text
<dataset_path>/
├── images/                 # 图像文件（与标注中的文件名一致）
├── annotations/            # 可选：每张图一个 JSON（X-AnyLabeling 单图导出）
│   └── *.json
├── vqa_labels.jsonl        # 可选：批量 JSONL
└── questions.txt           # 可选：旧版 CSV 风格（向后兼容）
```

若三类标注都不存在或无法解析出样本，会报错：`No valid data found in <path>`。

### 格式 1：`annotations/*.json`（X-AnyLabeling 单图）

- 每个 JSON 对应一张图；通过字段 **`imagePath`** 与 **`images/`** 下文件关联（代码中为 `data_path / 'images' / imagePath`）。
- 问答对放在 **`vqaData`**：键为问题字符串，值为答案（字符串或单元素列表）。

示例（字段名需与导出工具一致）：

```json
{
  "imagePath": "sample_001.jpg",
  "imageWidth": 640,
  "imageHeight": 480,
  "version": "x-anylabeling",
  "vqaData": {
    "Is the person wearing a helmet?": "yes",
    "是否有遮挡？": "否"
  }
}
```

### 格式 2：`vqa_labels.jsonl`（批量）

每行一个 JSON 对象：

- **`image`**：相对 `images/` 的文件名。
- 可选 **`width` / `height`**。
- 其余键视为「问题 → 答案」；答案可为字符串或列表（取首元素）。

示例：

```json
{"image": "a.jpg", "width": 224, "height": 224, "Is it red?": "yes"}
{"image": "b.jpg", "Contains text?": ["no"]}
```

### 格式 3：`questions.txt`（向后兼容）

每行逗号分隔，至少 7 列：

`image_name, x, y, w, h, question, answer`

- `x,y,w,h` 为整型 bbox；加载时会**按 bbox 从整图裁剪 ROI** 再送入网络（与仅使用 ROI 整图的路径不同）。
- 图像路径为 `images/<image_name>`。

### 答案归一化与标签

- **`data.answer_mapping`**（`default.yaml`）中的 **`positive` / `negative`** 词表用于**子串匹配**（答案先 `strip().lower()` 再匹配）。
- 匹配到肯定 → 内部 **`YES`**；匹配到否定 → **`NO`**；都未命中 → 使用 **`default`**（默认映射为 `'NO'`，再参与同一套规则）。
- 训练时：**`YES` → 标签 1，`NO` → 标签 0**（`datasets.py` 中 `label`）。

自定义领域词时，在 YAML 中扩展 `positive` / `negative` 列表即可。

### 数据划分与 DataLoader

- **`data.split`**：`train_ratio` / `val_ratio` / `test_ratio`（默认 0.7 / 0.2 / 0.1）。**一键训练**当前使用 `split_dataset` 得到 train / val，**test 集在一键脚本中未接入 DataLoader**（若需固定测试集评估，需在代码中接 `test_ds`）。
- **`data.dataloader`**：`batch_size`、`num_workers`、`pin_memory` 等；验证集 dataloader 使用 `split="val"`（增强关闭逻辑见 `create_dataloader`）。

## 配置说明（`default.yaml` 摘要）

配置文件默认路径由包内 **`get_builtin_default_config_path()`** 指向 **`speedvqa/configs/default.yaml`**。建议**复制为新文件**再改路径与实验名，避免污染默认文件。

| 节点 | 作用 |
|------|------|
| **`model`** | `vision.backbone`（如 `mobilenet_v3_small`、`resnet50`）、`text.encoder`、`fusion`、`classifier`、`loss` |
| **`data`** | `dataset_path`、`image`（尺寸与归一化）、`augmentation`、`dataloader`、`split`、`answer_mapping` |
| **`train`** | `epochs`、`save_dir`、`experiment_name`、`optimizer`、`scheduler`、`strategy`（混合精度、early stopping）、`validation`、`logging` |
| **`train.resume`** | 非空时从该 checkpoint 继续训练（需与当前模型结构一致） |
| **`val`** | 验证 batch、是否保存预测、`save_dir` |
| **`inference`** | `device`、`batch_size`、后处理阈值等（一键预测若未传 `--config`，PyTorch 会从 checkpoint 内嵌 config 推断 tokenizer 相关项） |
| **`hardware`** | `gpu.device_ids`、`gpu.memory_fraction`、`cpu.num_threads` |

**检查点目录**：`train.save_dir` 宜落在 `runs/` 下；若写成仓库根 `.` 等，**`speedvqa.utils.artifact_paths`** 可能把写入路径改写到 `runs/train/<experiment_name>/`（见 `trainer` 日志提示）。

## 训练

### 命令行（推荐在仓库根执行）

```bash
cd /path/to/SpeedVQA

# 使用包内默认配置
python -m speedvqa.cli.onekey_train

# 指定配置与数据根目录（覆盖 data.dataset_path）
python -m speedvqa.cli.onekey_train --config /path/to/my_exp.yaml --data /path/to/my_dataset

# 等价：speedvqa-train（安装 entry points 后）
speedvqa-train --config ./my_exp.yaml --data ./datasets/vqa_dataset
```

| 参数 | 含义 |
|------|------|
| `--config` | YAML 路径；默认包内 `default.yaml` |
| `--data` | 覆盖 **`data.dataset_path`** |

### Shell 包装

**`scripts/onekey_train.sh`**：`cd` 到仓库根后执行 `python3 -m speedvqa.cli.onekey_train "$@"`，故可传同样参数：

```bash
./scripts/onekey_train.sh --config ./my_exp.yaml --data ./datasets/vqa_dataset
```

### 训练产物

默认在 **`train.save_dir`** 下以 **`experiment_name`** 为子目录（并受 artifact 规范化影响），常见文件包括：

- `latest_checkpoint.pth`、`best_checkpoint.pth`
- 周期性：`checkpoint_epoch_N.pth`
- 日志与 TensorBoard（若开启）

Checkpoint 内宜含 **`model_state_dict`** 与完整 **`config`**（一键导出依赖后者；若缺失需导出时显式传 `--config`）。

## 预测（验证链路）

对**单张 ROI 图**与**一个问句**做一次推理，打印 `answer`（YES/NO）、`confidence`、`time_ms`。

```bash
python -m speedvqa.cli.onekey_predict \
  --model runs/train/speedvqa_exp1/checkpoints/best_checkpoint.pth \
  --image path/to/roi.jpg \
  --question "Is the object damaged?" \
  --format pytorch \
  --device cuda
```

| 参数 | 含义 |
|------|------|
| **`--model`** | PyTorch 可为训练保存的 **checkpoint**（内含权重）；ONNX/TensorRT 为对应 `.onnx` / `.engine` |
| **`--image`** | 图像路径（应为 ROI；与训练时「整图+无 bbox」或「questions.txt 裁剪」保持一致） |
| **`--question`** | 问句文本 |
| **`--format`** | `pytorch` \| `onnx` \| `tensorrt` |
| **`--device`** | `cuda` 或 `cpu`（无 GPU 时会落到 CPU） |
| **`--config`** | 可选 YAML；不传且 `format=pytorch` 时，从 checkpoint 的 **`config`** 推断 `image_size`、`text_encoder`、`max_length`、阈值等 |

## 模型导出

一键导出从 checkpoint 恢复 **`config`** 与权重，再调用 **`export_model`**（见 **`speedvqa/export/exporter.py`**）。

```bash
cd /path/to/SpeedVQA

python -m speedvqa.cli.onekey_export \
  --checkpoint runs/train/speedvqa_exp1/checkpoints/best_checkpoint.pth \
  --output exports \
  --name speedvqa_export \
  --formats pytorch,onnx
```

| 参数 | 含义 |
|------|------|
| **`--checkpoint`** | 训练生成的 `.pth` |
| **`--config`** | 可选；**默认使用 checkpoint 内嵌 `config`**；若无内嵌 config 则**必填** |
| **`--output`** | 输出目录（相对于**当前工作目录**，一般在仓库根执行即为 `仓库根/exports`） |
| **`--name`** | 文件基名（**不含扩展名**） |
| **`--formats`** | 逗号分隔：`pytorch`、`onnx`、`tensorrt`（TensorRT 依赖 ONNX，逻辑上会先确保 ONNX 可用） |

**生成文件路径**（`base_path = output_dir / name`）：

- PyTorch：**`<output>/<name>.pt`**
- ONNX：**`<output>/<name>.onnx`**
- TensorRT：**`<output>/<name>.engine`**

导出结束后终端会打印各格式大小、耗时及校验摘要。更复杂的校验与基准见 **`speedvqa/examples/model_export_example.py`**、**`performance_benchmark_example.py`**。

## 部署与推理

### 使用 `ROIInferencer`（Python）

构造参数与 **`onekey_predict`** 一致：`model_path`、`model_format`、`device`，以及 **`config`** 字典（键建议与 CLI 推断一致）：

- `image_size`：`(H, W)`，默认 `(224, 224)`
- `text_encoder`：与训练一致，如 `distilbert-base-uncased`
- `max_text_length`
- `confidence_threshold`

```python
from speedvqa.inference.inferencer import ROIInferencer

inferencer = ROIInferencer(
    "exports/speedvqa_export.onnx",
    model_format="onnx",
    device="cuda",
    config={
        "image_size": (224, 224),
        "text_encoder": "distilbert-base-uncased",
        "max_text_length": 128,
        "confidence_threshold": 0.5,
    },
)
result = inferencer.inference("roi.jpg", "Is it a defect?")
# result.answer, result.confidence, result.inference_time_ms, result.probabilities
```

批量场景用 **`batch_inference(roi_images, questions)`**（两列表长度须一致）。细节与边界情况见 **`speedvqa/inference/README.md`**。

### 部署检查清单

1. **预处理一致**：输入应为与训练相同的 ROI 语义；归一化与 `image_size` 与配置一致。
2. **文本编码器**：推理机需能加载与训练相同的 **`text_encoder`**（HF 缓存或离线拷贝）。
3. **ONNX**：CPU 部署可使用 `CPUExecutionProvider`；GPU 使用 `CUDAExecutionProvider`。
4. **TensorRT**：引擎与 GPU 架构、CUDA、TensorRT 版本强相关，需在目标环境生成或迁移兼容的 `.engine`。

## `runs/` 输出目录

统一在 **`runs/`** 管理训练、验证、推理、导出与超参搜索等输出（具体子目录依实验脚本与配置而定）。

**路径约束**：若配置把检查点写到非约定位置，**`speedvqa.utils.artifact_paths`** 可能改写为 `runs/train/<experiment_name>/` 或 `exports/<experiment_name>/`。实现见 **`speedvqa/utils/artifact_paths.py`**。

### 目录结构（约定）

```text
runs/
├── train/{experiment_name}/
│   ├── checkpoints/
│   ├── logs/
│   └── tensorboard/
├── val/
│   ├── predictions/
│   └── metrics/
├── inference/
│   ├── results/
│   └── visualizations/
├── export/
│   ├── pytorch/
│   ├── onnx/
│   └── tensorrt/
└── hyperopt/{timestamp}/
    ├── trials/
    └── summary.json
```

### 配置片段示例

**训练**（`default.yaml`）：

```yaml
train:
  save_dir: './runs/train'
  experiment_name: 'speedvqa_exp1'
```

**验证**：

```yaml
val:
  save_dir: './runs/val'
  save_predictions: true
```

### 清理

```bash
./scripts/onekey_clean.sh --dry-run
./scripts/onekey_clean.sh
```

或按需删除 `runs/` 子目录；**`runs/` 不应提交**（已在 `.gitignore`）。

### 实践建议

- 实验名带时间戳便于追溯，例如 `speedvqa_20260329_143000`。
- 重要检查点备份到仓库外；每次实验可拷贝 YAML 到 `runs/train/<experiment_name>/config.yaml`。
- 大磁盘：`ln -s /path/to/large/disk/runs ./runs`。

## 常见问题（简要）

| 现象 | 可能原因 |
|------|----------|
| `No valid data found` | `dataset_path` 下无 `images/` 或三种标注均为空/路径错 |
| `Image not found` | JSON/JSONL 中文件名与 `images/` 内不一致 |
| 导出报缺少 config | checkpoint 非本流程保存或无 `config` 字段，需 **`--config`** |
| ONNX/TensorRT 失败 | 未安装依赖，或 CUDA/TRT 版本不匹配 |
| 推理很慢 | 首次加载 HF tokenizer/权重；可预热若干次；TensorRT/ONNX 相对纯 PyTorch 在部分环境更快 |

## 开发补充：目录入口、导入与测试

### 目录与 CLI

- **包代码**：`speedvqa/`（`data`、`models`、`engine`、`inference`、`export` 等）。
- **一键 CLI**：`onekey_train` / `onekey_predict` / `onekey_export`；安装后为 **`speedvqa-train`** 等。
- **Shell**：仓库根 **`scripts/onekey_*.sh`**；清理 **`scripts/onekey_clean.sh`**。
- 公开 API 以 **`speedvqa/__init__.py`** 的 **`__all__`** 为准。

### 推荐导入

| 用途 | 推荐方式 |
|------|----------|
| 数据集、YAML、从配置构建模型 | `from speedvqa import ...` 或 `speedvqa.data` / `speedvqa.utils.config` / `speedvqa.models` |

**`pip install` 生成的 wheel 不包含** `speedvqa.tests` 与 `speedvqa.examples`（见 **`pyproject.toml`** 的 `exclude`）；克隆源码开发时二者完整可用。
