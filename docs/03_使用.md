# 03 使用（全栈流程）

本文档覆盖从 **环境 → 数据 → 训练 → 导出 → 部署推理** 的完整链路；**架构与设计理念**见 **[01_设计.md](./01_设计.md)**。模块级 API 细节见 **`speedvqa/inference/README.md`**、**`speedvqa/export/README.md`**。

---

## 全栈流程概览

1. **环境与安装** → 2. **数据准备** → 3. **配置** → 4. **训练** → 5. **（可选）预测验证** → 6. **导出** → 7. **部署推理**  
   产物与日志默认落在 **`runs/`**（见下文「`runs/` 输出目录」）。

---

## 环境与安装

在仓库根目录（例如 `/path/to/SpeedVQA`）执行：

```bash
pip install -r requirements.txt
pip install -e .
```

可编辑安装便于修改 `speedvqa/` 后立即生效。部署前请确认驱动、CUDA、TensorRT（若使用）与 **`requirements.txt`** 版本匹配。

---

## 数据准备

- **任务形态**：ROI 裁剪图 + 短问句，**二分类**（是/否）；标注优先兼容 **X-AnyLabeling** 导出（单图 JSON、`annotations/*.json`、`vqa_labels.jsonl` 等，具体以 `speedvqa/data` 实现为准）。
- **数据集根目录**：与 **`speedvqa/configs/default.yaml`** 中 **`data.dataset_path`** 一致；其下按约定放置 `annotations/`、`vqa_labels.jsonl` 等。
- **答案归一化**：在 YAML 中配置 **`data.answer_mapping`**，将多种中英文肯定/否定表述映射到统一二分类标签。
- 设计背景与格式要点见 **[01_设计.md](./01_设计.md)**「配置与数据要点」「典型工作流」。

---

## 配置

- 复制或修改 **`speedvqa/configs/default.yaml`**：`model`、`data`、`train`、`export`、`val` 等节点控制骨干、batch、轮数、保存路径与实验名等。
- 完整键说明以该文件内注释为准；架构级说明见设计文档。

---

## 训练

- **Python**：`python -m speedvqa.cli.onekey_train`  
- **已安装包**：**`speedvqa-train`**（见根目录 **`pyproject.toml`** 的 **`[project.scripts]`**）  
- **Shell**：仓库根 **`scripts/onekey_train.sh`**（内部调用上述模块）

训练检查点、日志默认写入 **`runs/train/<experiment_name>/`**（见下文）。

---

## 预测（验证链路）

- **`python -m speedvqa.cli.onekey_predict`** 或 **`speedvqa-predict`**  
- **`scripts/onekey_predict.sh`**

---

## 模型导出

- **Python**：**`python -m speedvqa.cli.onekey_export`** 或 **`speedvqa-export`**  
- **Shell**：**`scripts/onekey_export.sh`**  
- 程序化导出： **`ModelExporter`**；示例：**`speedvqa/examples/model_export_example.py`**  
- 格式与配置项：**`speedvqa/export/README.md`**

---

## 部署与推理

- 使用 **`ROIInferencer`** 加载与导出格式一致的权重（PyTorch / ONNX / TensorRT），在目标 GPU 上验证延迟与准确率。  
- API、批推理与参数：**`speedvqa/inference/README.md`**  
- 性能与压测可参考 **`speedvqa/examples/`** 下相关脚本。

## `runs/` 输出目录

统一在 **`runs/`** 管理训练、验证、推理、导出与超参搜索等输出。

**路径约束**：在仓库根运行时，若配置会把检查点写到非约定位置，`speedvqa.utils.artifact_paths` 会将目录规范到 `runs/train/<experiment_name>/` 或 `exports/<experiment_name>/`（`runs/`、`exports/`、`cache/` 已在 `.gitignore`）。实现见 **`speedvqa/utils/artifact_paths.py`**。

### 目录结构（约定）

```
runs/
├── train/{experiment_name}/
│   ├── checkpoints/
│   ├── logs/
│   └── tensorboard/
├── val/
│   ├── predictions/
│   └── metrics/
├── inference/
│   ├── results/
│   └── visualizations/
├── export/
│   ├── pytorch/
│   ├── onnx/
│   └── tensorrt/
└── hyperopt/{timestamp}/
    ├── trials/
    └── summary.json
```

### 配置片段示例

**训练输出**（`default.yaml`）：

```yaml
train:
  save_dir: './runs/train'
  experiment_name: 'speedvqa_exp1'
```

训练过程常见文件：`latest_checkpoint.pth`、`best_checkpoint.pth`、`checkpoint_epoch_N.pth`、`training.log`。

**验证输出**：

```yaml
val:
  save_dir: './runs/val'
  save_predictions: true
```

**导出路径示例**（Python）：

```python
exporter.export_pytorch(model, 'runs/export/pytorch/model.pt')
exporter.export_onnx(model, 'runs/export/onnx/model.onnx')
exporter.export_tensorrt(model, 'runs/export/tensorrt/model.engine')
```

超参优化结果位于 `runs/hyperopt/{timestamp}/`（如 `trial_*.json`、`optimization_summary.json` 等）。

### 清理

```bash
./scripts/onekey_clean.sh --dry-run
./scripts/onekey_clean.sh
```

或按需删除 `runs/` 子目录；**`runs/` 不应提交**（已在 `.gitignore`）。

### 实践建议

- 实验名带时间戳便于追溯，例如 `speedvqa_20260329_143000`。  
- 重要检查点备份到仓库外；每次实验可拷贝一份 YAML 到对应 `runs/train/<experiment_name>/config.yaml`。  
- 若需大磁盘：可将 `runs` 指到外挂路径，例如 `ln -s /path/to/large/disk/runs ./runs`。

## 开发补充：目录入口、导入与测试

### 目录与 CLI

- **包代码**：`speedvqa/`（`data`、`models`、`engine`、`inference`、`export` 等）。  
- **一键 CLI**：`onekey_train` / `onekey_predict` / `onekey_export`；**`pip install -e .`** 后为 **`speedvqa-train`**、**`speedvqa-predict`**、**`speedvqa-export`**。  
- **Shell**：仓库根 **`scripts/onekey_*.sh`**；清理 **`scripts/onekey_clean.sh`**。  
- 公开 API 以 **`speedvqa/__init__.py`** 的 **`__all__`** 为准。

### 推荐导入

| 用途 | 推荐方式 |
|------|----------|
| 数据集、YAML、从配置构建模型 | `from speedvqa import ...` 或 `speedvqa.data` / `speedvqa.utils.config` / `speedvqa.models` |
| 训练、超参搜索 | `from speedvqa.engine import train, ConfigurableTrainer` 等（见 `speedvqa/engine/__init__.py`） |
| 导出 | `speedvqa.export` |
| ROI 推理 | `speedvqa.inference` |

**`pip install` 生成的 wheel 不包含** `speedvqa.tests` 与 `speedvqa.examples`（见 **`pyproject.toml`** 的 `exclude`）；克隆源码开发时二者完整可用。

### 测试

在仓库根：

```bash
pytest
```

或：

```bash
python speedvqa/tests/run_tests.py
```

配置见根目录 **`pyproject.toml`**（`testpaths`）与 **`speedvqa/pytest.ini`**；Hypothesis、`collect_ignore` 见 **`speedvqa/tests/conftest.py`**。GPU / TensorRT 相关测试前请确认环境与依赖一致。
