## VisionJudge 2.0 全功能文档

### 1️⃣ 项目简介与核心价值

#### 1.1 产品概述
VisionJudge 2.0是一款工业级视觉智能判官系统，它采用创新的"一个问题 + 一个区域/Crop + 一个小模型 = 一个新算法"核心理念，能够用同一套架构、同一套模型、同一套流水线，高效实现全行业150+项视觉AI算法需求。

#### 1.2 核心价值
- **工业级优化**：经过实战验证的算法性能与稳定性
- **低成本标注**：每类算法仅需50~100张标注图
- **高速推理**：平均推理速度<50ms，高密度场景<200ms
- **位置可输出**：100%支持框/区域/热力图输出
- **跨场景复用**：同一模型，只需换问题模板即可支持多场景
- **边缘部署友好**：支持树莓派5/Jetson Nano/手机端等低算力设备

#### 1.3 市场定位
VisionJudge 2.0填补了工业视觉领域通用解决方案的空白，为工厂、工地、社区、零售、电力、交通等多个行业提供了一站式视觉AI能力平台，可以快速响应不同场景的定制化需求，大幅降低算法开发和部署成本。

---

### 2️⃣ 🏗️ 整体架构设计

#### 2.1 系统架构概述
VisionJudge 2.0采用创新的流水线架构设计，实现了从图像输入到结果输出的全流程优化：

**主要流程**：通用检测器 → 上下文Crop → 轻量预筛模型 → 空间聚类 → 批处理推理

这一架构设计使得系统能够高效处理各种复杂场景，同时保持极低的资源占用和高速的推理速度。

#### 2.2 核心组件
- **通用检测器**：负责初步识别图像中的关键目标
- **上下文Crop工具**：根据不同算法需求，智能裁剪目标区域
- **轻量预筛模型**：快速过滤掉明显不符合条件的区域，减少计算量
- **空间聚类模块**：对相似区域进行聚类，优化批处理效率
- **批处理推理引擎**：并行处理多个区域，大幅提升整体性能

#### 2.3 技术栈
- 主模型：DINOv2 + TinyLM组合架构
- 预筛模型：轻量级CNN模型(<0.1M参数)
- 空间聚类：优化版DBSCAN算法
- 推理框架：支持ONNX Runtime、Torch等多种推理后端
- 部署选项：Docker容器、边缘设备、云服务器

### 3️⃣ 🧠 核心算法详解

#### 3.1 按技术实现方式分类

##### 3.1.1 👤 人员身份与行为类
- **人脸识别**：支持专用人脸模型或VisionJudge微调
- **黑名单/轨迹追踪**：ID绑定+时序跟踪
- **年龄检测**：人脸Crop + 多分类头
- **躺坐识别**：上下文Crop + 姿态语义，已专项优化
- **签到/支付**：人脸+区域绑定

**实现方式**：人脸/人体Crop + VisionJudge问答（"是否躺坐？"、"是否黑名单？"）

##### 3.1.2 🧥 着装与防护合规类
- **安全帽、工服、反光衣、厨师帽、口罩、长袖、绝缘手套、裸露皮肤、安全带识别**

**实现方式**：局部Crop（头部/上半身） + 二分类问答
- 标注成本：每类仅需50~100张正负样本
- 位置输出：返回对应部位框（如"未戴安全帽的人头位置"）

##### 3.1.3 🐀 环境生物监测类
- **厨房夜间老鼠识别**

**实现方式**：整图VQA模式 + 区域聚类验证
- 支持红外/低光增强
- 输出老鼠出现区域框

##### 3.1.4 🚫 违规行为识别类
- **抽烟、打电话、离岗、睡岗、翻越、打架、徘徊、占道经营、餐饮出店识别**

**实现方式**：
- 抽烟/打电话 → 手部+嘴部/耳部上下文Crop
- 离岗/睡岗 → 区域无人检测 + 时间序列
- 翻越 → 人体+栏杆上下文Crop + 姿态判断
- 占道/出店 → 区域聚类 + 场景级问答（"该区域是否在经营？"）
- 全部支持位置输出

##### 3.1.5 🚨 安全风险预警类
- **烟火、摔倒、危险闯入、爆破警戒、皮带异物、拥堵踩踏、物品遗留识别**

**实现方式**：
- 烟火 → 整图VQA + 局部验证
- 摔倒 → 人体姿态上下文Crop
- 物品遗留 → 背景差分 + 区域VQA
- 皮带异物 → 传送带区域固定Crop + "是否有异物？"
- 全部支持告警区域定位

##### 3.1.6 🚗 车辆与交通管理类
- **车牌、车型特征、违停、进出、电动车进电梯、渣土车、机非人、交通流、事件检测、共享单车违停识别**

**实现方式**：
- 车牌/车型 → 车辆Crop + OCR/分类头
- 违停/进电梯 → 区域聚类 + "是否在禁停区？"
- 渣土车 → 车辆Crop + 多分类
- 支持车牌/车辆框输出

##### 3.1.7 🏗️ 设备与设施状态类
- **油污、异物、箱体、空开、压板、旋钮、指示灯、灭火器缺失、传送带偏离识别**

**实现方式**：固定区域Crop + 状态问答
- 例："灭火器是否缺失？"、"指示灯是否亮？"
- 标注极简：只需标注"有/无"
- 输出设备区域框

##### 3.1.8 🌳 环境与空间管理类
- **垃圾堆放、裸土、通道占用、消防门堵塞、道路积水、户外广告牌、走廊管控识别**

**实现方式**：
- 垃圾/裸土/积水 → 整图或地面区域VQA
- 消防门堵塞 → 门区域Crop + "是否被堵？"
- 广告牌 → 区域Crop + OCR/合规判断
- 输出问题区域框

##### 3.1.9 👥 人流与密度分析类
- **人数统计、密度、客流、聚集、排队识别**

**实现方式**：
- 人数/密度 → 检测器计数 + 热力图
- 聚集 → DBSCAN聚类 + 区域问答
- 排队 → 线性区域密度分析
- 输出热力图/聚集区域框

##### 3.1.10 🚪 出入口与门禁类
- **陌生人识别、门禁联动、婴儿车/大件行李检测**

**实现方式**：
- 陌生人 → 人脸比对 + VisionJudge"是否注册？"
- 婴儿车/行李 → 物体检测 + 区域关系判断
- 支持联动门禁系统

### 4️⃣ ⚡ 性能优化策略

#### 4.1 模型压缩与加速
- **轻量级预筛模型**：<0.1M参数，快速过滤无效区域
- **批处理推理**：智能合并相似区域，大幅提升吞吐量
- **ONNX导出优化**：支持动态批处理和量化推理
- **硬件加速支持**：针对不同硬件平台的算子优化

#### 4.2 空间聚类优化
- **自适应DBSCAN**：根据场景自动调整聚类参数
- **区域合并策略**：智能合并相邻的相似区域
- **层次化聚类**：先粗聚类再细分类，提升处理效率

#### 4.3 算法级优化
- **上下文感知Crop**：只关注最相关的区域，减少计算量
- **时序分析缓存**：利用历史信息，避免重复计算
- **渐进式推理**：从低精度快速筛选，必要时才用高精度模型
- **模型蒸馏**：将大模型能力迁移到小模型

#### 4.4 性能测试结果

| 指标                | VisionJudge 2.0 表现              |
| ------------------- | --------------------------------- |
| 新增算法成本        | 50~100 张标注图 + 1小时训练       |
| 推理速度（单任务）  | < 50ms（平均）                    |
| 高密度场景（100人） | < 200ms（预筛+聚类+批处理）       |
| 位置输出            | ✅ 100% 支持（框/区域/热力图）     |
| 跨行业复用          | ✅ 同一模型，只需换问题模板        |

| 标注难度            | 极低（区域级标注比个体标注更省）  |

### 5️⃣ 🚀 部署与应用指南

#### 5.1 部署选项
VisionJudge 2.0支持多种部署方式，适应不同的应用场景和硬件环境：

##### 5.1.1 云服务器部署
```python
# 云服务器部署示例（Python API调用）
import requests

url = "http://your-server.com/api/v1/visionjudge/infer"
payload = {
    "image": "base64_encoded_image",
    "question": "是否戴安全帽？"
}
headers = {
    "Content-Type": "application/json",
    "Authorization": "Bearer your_token"
}

response = requests.post(url, json=payload, headers=headers)
result = response.json()
print(result["answer"])
print(result["bbox"])
```

##### 5.1.2 边缘设备部署
```bash
# 树莓派部署示例
# 安装依赖
pip install torch torchvision onnxruntime opencv-python

# 运行推理
python infer_edge.py --model visionjudge.onnx --input rtsp://camera --question "是否戴安全帽？"
```

##### 5.1.3 Docker容器部署
```bash
# Docker部署示例
# 拉取镜像
docker pull visionjudge/v2:latest

# 运行容器
docker run -d -p 8000:8000 --name visionjudge \
  -v /path/to/models:/app/models \
  -v /path/to/configs:/app/configs \
  visionjudge/v2:latest
```

#### 5.2 API设计
VisionJudge 2.0提供了简洁而强大的HTTP/REST接口，支持图像和视频流输入：

##### 5.2.1 单图像推理接口
```
POST /api/v1/visionjudge/infer
Content-Type: application/json

{
  "image": "base64_encoded_image",
  "question": "是否戴安全帽？",
  "threshold": 0.8
}

# 响应示例
{
  "success": true,
  "answer": 1,
  "confidence": 0.95,
  "bbox": [100, 150, 80, 120],
  "time_ms": 45
}
```

##### 5.2.2 视频流处理接口
```
POST /api/v1/visionjudge/stream
Content-Type: application/json

{
  "rtsp_url": "rtsp://camera:554/stream",
  "question": "是否有人员聚集？",
  "interval": 1.0,
  "alarm_callback": "http://your-server.com/alarm"
}

# 响应示例
{
  "success": true,
  "stream_id": "abc123",
  "status": "processing"
}
```

### 7️⃣ 🖼️ 标注规范与数据模板

#### 7.1 标注格式
VisionJudge 2.0采用COCO-like JSON格式进行数据标注，支持区域级和目标级标注：

```json
{
  "images": [{"id": 1, "file_name": "img1.jpg"}],
  "annotations": [
    {
      "id": 1,
      "image_id": 1,
      "bbox": [x,y,w,h],
      "category_id": 1,
      "answer": 1,  // 0=no, 1=yes
      "question": "是否戴安全帽？"
    }
  ],
  "categories": [{"id": 1, "name": "person"}]
}
```

#### 7.2 标注工具推荐
- **LabelImg**：适合个体框标注，操作简单直观
- **LabelMe**：支持区域/多边形标注，适合复杂形状标注
- **CVAT**：支持团队协作和视频标注，适合大规模标注项目

#### 7.3 标注指南
- **区域级标注**：对于场景级算法，只需标注关注区域和整体判断结果
- **目标级标注**：对于需要定位的算法，需标注具体目标位置和判断结果
- **正负样本比例**：建议1:1~1:3，确保模型泛化能力
- **标注精度要求**：位置误差不超过5%，分类准确率不低于95%

#### 7.4 数据模板
VisionJudge 2.0提供了丰富的数据模板，涵盖各行业常见场景：

- **安全帽识别模板**：包含不同角度、光线条件下的戴帽/未戴帽样本
- **着装合规模板**：包含各种工服、反光衣、防护装备的样本
- **车辆管理模板**：包含各种车型、车牌、违停场景的样本
- **环境监测模板**：包含烟火、积水、垃圾等异常场景的样本

#### 10.2 快速入门示例

```python
# infer.py 示例代码
from models.vision_judge import VisionJudge
import cv2

# 初始化模型
vj = VisionJudge(model_path="models/visionjudge.onnx")

# 加载图像
image = cv2.imread("test.jpg")

# 单问题推理
result = vj.infer(image, question="是否戴安全帽？")
print(f"答案: {result['answer']}")
print(f"置信度: {result['confidence']}")
print(f"位置: {result['bbox']}")

# 可视化结果
vis_image = vj.visualize(image, result)
cv2.imwrite("result.jpg", vis_image)

# 多问题批量推理
results = vj.batch_infer(image, questions=[
    "是否戴安全帽？",
    "是否穿工服？",
    "是否抽烟？"
])
for i, res in enumerate(results):
    print(f"问题{i+1}: {res['question']}")
    print(f"答案{i+1}: {res['answer']}")
```
